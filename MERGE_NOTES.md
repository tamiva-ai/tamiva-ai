# v39 merge notes

This zip was generated by applying the v38 admin-portal changes
(`POST /admin/logs`, `GET /admin/client-logs`, `ProviderCall` /
`ClientLog` tables, the updated admin.html) on top of the working
build, then incrementally fixing compile errors as the build surfaced
them.

## Files where the v39 zip diverges from the working build

`flutter_app/lib/screens/brand_assets_screen.dart` is a *rewrite*
of the working build's version. The working build's file (md5
`b1933096…`) was broken in three places:

1. The build call site at L628 used a slim ternary chain
   (`_logoLockedBehindSupport ? const _LogoSupportLockTile() : …`)
   that referenced symbols that had been removed in earlier work
   (`_logoReady`, `_logoLockedBehindSupport`, `_project`,
   `_beginLogoGeneration`, `_projectId`, `_logoAttempts`,
   `_startingLogo`, `_generationStartedAt`, `_logoMaxAttempts`,
   `openProjectPreview`, etc.). The Dart compiler reported this as
   "Can't find ')' to match '('", "Too many positional arguments",
   and "The getter '_logoReady' isn't defined".
2. The `_BrandKitSection` class was missing the `showFreePill`
   parameter that the Website tile's call site passes.
3. The `_LogoPreviewState._startGenerationAttempt` method passed
   `projectId` (a `String?` at that point) directly into the
   `Project(id: projectId, …)` constructor. Dart's flow analysis
   doesn't propagate the `if (projectId == null) return;` null
   narrowing through the `setState` callback, so this is a hard type
   error.

To unblock the build, the v39 zip rewrites
`brand_assets_screen.dart` with:

- A slim `_BrandAssetsScreenState` (just `_tier`, `_refreshTier`,
  `_isPaid`, the build(), and `_openPricingScreen()`) — no more
  inline state machine.
- A self-contained `_LogoPreview extends StatefulWidget` +
  `_LogoPreviewState` mirroring the `_CarouselPreview` pattern:
  bootstrap on mount, adopt-on-rehydrate, in-flight via
  `setState`, poll every 3s, lock-on-success, retry-cap with
  support-only fallback after 3 failures.
- New helpers: `_LogoPlaceholder`, `_LogoReadyPreview`,
  `_LogoFailedTile`, `_LogoSupportLockTile`, `startLogoGeneration`.
- The three bug fixes: `showFreePill: showFreePill` on the
  `CascadedStack` call in `_BrandKitSection`, the `ExitOnBackScope`
  import, and a hoisted `final pid = projectId;` so the Dart
  analyzer is happy inside the `setState` callback.

If your parallel branch also has a `brand_assets_screen.dart`
rewrite, the merge will conflict. The semantic surface is
identical — only the internal organization changed.
